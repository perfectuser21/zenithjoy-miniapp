# ZenithJoy 小程序灰度发布方案

**版本**: 1.0
**日期**: 2026-04-12
**状态**: 锚定

---

## 一、灰度目标

KR3 目标：微信小程序上线，基础功能可用，无重大 bug。

灰度策略：先体验版内测 → 小范围开放（体验码分发）→ 全量上线。

---

## 二、灰度阶段划分

### 阶段 0：开发自测（当前）
- **用户范围**: 开发者账号（1-2 人）
- **工具**: 微信开发者工具 + 手机扫码预览
- **目标**: 核心功能 bug 清零，4 大 Tab 可正常打开
- **准入条件**: 本 PR 合并后即开始

### 阶段 1：内测（体验版）
- **用户范围**: 5-10 人（团队成员 + 核心用户）
- **工具**: 微信开发者工具 → "上传" → 设置为体验版
- **内测用户池**: 在微信小程序后台「成员管理」中逐一添加（微信号或手机号）
- **持续时间**: 3-5 天
- **目标**: 发现真实设备兼容性问题、UI 异常、网络超时

### 阶段 2：小范围开放
- **用户范围**: 受邀用户（朋友圈/私信分享体验码）
- **控制手段**: 不做微信官方审核提交，通过体验版体验码二维码分发
- **持续时间**: 5-7 天
- **目标**: AI 对话、支付、会员流程完整跑通

### 阶段 3：提交审核 + 全量上线
- **条件**: 阶段 2 无 P0/P1 bug，支付联调通过（需商户号配置）
- **操作**: 微信开发者工具 → 上传 → 微信公众平台提交审核

**路径**: 微信公众平台 → 小程序 → 成员管理 → 体验成员

**当前内测用户**:
| 序号 | 角色 | 备注 |
|------|------|------|

| 2-10 | 待添加 | 上线前补充 |

**添加方式**: 微信号搜索添加，或扫开发者工具中的「体验版二维码」

---

## 四、回滚策略

### 快速回滚（版本降级）
微信小程序后台 → 版本管理 → 选择历史版本 → 回滚

- 回滚决策条件: P0 级 crash / 支付失败率 > 20% / AI 服务完全不可用
- 回滚时间目标: < 15 分钟
- 回滚通知: 在内测用户群发送说明

### 云函数回滚
云函数独立部署，可单独回滚：
微信开发者工具 → 云开发控制台 → 云函数 → 历史版本

---

## 五、监控指标

| 指标 | 阈值 | 触发动作 |
|------|------|----------|
| 小程序崩溃率 | > 5% | 立即回滚 |
| AI 调用超时率 | > 30% | 检查云函数日志 |
| 支付失败率 | > 20% | 回滚 + 排查商户号配置 |
| 页面加载时间 | > 3s | 优化图片/接口 |

**监控工具**: 微信公众平台 → 小程序 → 数据分析 → 运营数据

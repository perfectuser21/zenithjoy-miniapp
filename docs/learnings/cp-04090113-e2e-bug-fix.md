# Learning: miniapp 端到端集成验证 Bug 修复

**分支**: cp-04090113-e2e-bug-fix
**日期**: 2026-04-09

## 修复列表

### Bug 1: checkQuota 误阻断无限配额 premium 用户

### 根本原因
`remainingQuota = -1` 表示无限制，但 `remaining <= 0` 判断将 -1 视为耗尽，导致 premium 会员无法发送 AI 消息。

### 下次预防
- [ ] 任何涉及"配额计数"的逻辑，`-1` 的语义必须在条件里显式排除
- [ ] 写配额检查时加注释说明 `-1 = unlimited` 的约定

---

### Bug 2: WXML 中调用 Page JS 方法无效

### 根本原因
微信 WXML 模板不支持直接调用 Page 的 JS 方法（`{{formatDate(xxx)}}`），只能读取 `data` 中的值。`expireDate` 始终显示为空。

### 下次预防
- [ ] WXML 模板中禁止写函数调用，所有格式化逻辑在 `setData` 前完成
- [ ] 日期/金额等需要格式化展示的字段，在 setData 时同时写入 `xxxText` 计算属性

---

### Bug 3 & 4: 文章列表和详情云函数被手动禁用

### 根本原因
之前某次开发手动将两个云函数改为返回空/禁用状态（可能为了避免测试干扰），但没有还原，导致文章流程完全断路。

### 下次预防
- [ ] 禁用云函数要加醒目注释 `// TODO: 恢复` 并在 PR 描述里说明
- [ ] 端到端验证任务要优先检查云函数返回值是否被手动覆盖
